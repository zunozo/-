정보보희 기본 요소 및 목적
기밀성 : 인가된 사용자만이 데이터에 접근
무결성(Integrity) : 결함이 없도록 하는 성질, 즉 인가된 사용자만이 데이터를 수정
가용성 : 인가된 사용자가 운허ㅏ는 시간이나 장소에서 필요 정보에 접근할 수 있고, 사용할 수 있도록 보장하는 성질.
책임성 : 특정한 작업 또는 행위에 대한 책임 소재를 확인 가능함을 의미.
인증성 : 참과 거짓의 근거가 될 수 있는 무언가를 확인하는 성질.

XSS(Cross-Site Scripting) : 웹 페이지에 악의적인 스크립트를 포함시켜 사용자 측에서 실행되게 유도함으로써, 정보 유출 등의 공격을 유발할 수 있는지 검토함.
Pharming : daum.net을 가짜로 만들어서 이쪽으로 연결시킨뒤 개인정보유출
Phishing : 특정 웹사이트를 가짜로 개설.



DAC(Discreationary Access Control, 임의 접근 통제) : 사용자의 신원정보를 통해 권한의 부여 및 회수에 대한 메커니즘을 기반으로 함. ex) DBMS 의 REVOKE, GRANT

MAC(Mandatory Access Control, 강제적 접근 통제) : 시스템의 사용자들은 자신의 정보에 대한 어떠한 접근 권한도 설정할 수 없다.

RBAC(Role-Based Access Control, 역할 기반 접근 통제) : 특정 역할들을 정의하고 각 역할에 따라 접근 권한을 지정하고 제어하는 방식.
