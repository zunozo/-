CISC(complex instruction set computer)
많은수의 명령어를 가짐, 명령어의 길이가 길어짐, 명령어가 소프트웨어적이므로 호환성이 좋음, 가변길이 명령어 형식

RISC(reduced instruction set computer)
적은수의 명령어, 명령어의 길이가 짧아짐, 명령어가 하드웨어적임, 고정길이 명령어 형식.

DBMS 분석 시 고려사항
가용성(availability), 성능, 구축비용, 기술지원, 상호 호환성

WAS 종류
eclipse -> jetty

구조적 방법론
분할과 정복 방법으로 하향식으로 기능을 분해한다.
재사용성이 낮음
module 중심
정보공학 방법론
자료구조 중심, 생명주기를 이용해 대형프로젝트를 수행
객체지향 방법론
객체중심(캡슐화, 추상화 기술이 필요)
애자일 방법론
사용자의 요구사항이 우선시됨.
반복적인 개발을 통한 잦은 출시를 목표로함
테일러링 개발 방법론
프로젝트분석이 가장 중요.
표준이 없음. 안내서들만이 존재, 커스터마이징의 작업이 반복

XP(익스트림 프로그래밍)(애자리 방법론)
피드백, 용기, 존경, 의사소통, 단순함

비용 측정 방법론의 분류
하향식
전문가 측정, 델파이 측정

ISO 12207 표준의 기분 공정 프로세스 항목
기본 공정, 조직 공정, 지원 공정

CMMI의 프로세스 영역
프로세스 관리, 프로젝트 관리, 엔지니어링, 지원

SPICE 모델
소프트웨어의 품질 및 생산성 향상을 위해 소프트웨어프로세스를 평가 및 개선하는 국제 표준이다.
SPICE 모델의 프로세스 수행 능력 수준 6단계
불안정 -> 수행 -> 관리 -> 확립 -> 예측 -> 최적화( 이미 만들어진 것을 확인하기때문에, 분석X)

DBMS의 필수 기능
DDefinitionL(데이터 정의), DManifulationL(데이터 조작), DControlL(데이터 제어)

무결성 제약조건
도메인 무결성, 개체 무결성, 참조 무결성
개체 -> 키
참조 -> 외래키(참조할수 없는 외래키값을 가질수없다.)
