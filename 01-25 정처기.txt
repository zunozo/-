인조 식별자
최대한 범용적인 값을 사용한다.
편의성과 단순성을 확보하기위한 인조 식별자를 사용할 수 있음.

속성
더 이상 분리되지 않는 최소의 데이터단위.
필드에 해당.
자체만으로 정보를 표현할 수 없음.
하나의 속성은 하나의 개체 타입에만 존재할 수 있음.

개념 데이터 모델링 -> 논리 데이터 모델링 -> 물리데이터 모델링 -> db구축

논리적 데이터 모델링
정규화하여 모델링.

확장ERD
특수화, 일반화(서로반대)
특수화 : 무슨동물인지 특정하는것(Top-down)
일반화 : 어떤과인지 일반화하는것(Bottom-up)
상속
집단화 : 각각의 개체 타입으로 새로운 개체 타입을 만들 수 있음, 상속관계를 가질 수 없음
부분 전체로 기억(학교 -> 학생,교수)
is-part-of
분류화 : 개체 타입과 개체 타입의 개체 어커런스들 간의 관계를 표현할 수 있다.
is-member-of 관계 상속x
부서 -> 인사부, 생산부, 영어부

data dependency
Functional, Join, Multivalued

종속자
결정자의 값에 의해 정해지는 값

결정자
속성 간의 종속성을 규명할 때 기준이 되는 값.

직접종속 : 일반적인 함수종속
간접종속 : 이행적 함수종속 분리해야함.

제1정규형 : 원자적
제2정규형 : 완전함수종속
제3정규형 : 이행함수종속
BCNF : 결정자가 후보키가 아닌 함수종속 제거
제4정규형 : 다치어트리뷰트 제거
제5정규형 : 후보키를 통하지 않은 조인종속제거

데이터 모델의 품질 기준
정확성
완전성
준거성
최신성
일관성
활용성 : 모두에게 의미를 충분하게 전달가능

관계스키마 설계 원칙
개체, 속성, 관계를 식별하여 릴레이션을 구성한다.
속성 사이의 관계와 데이터의 종속성을 고려하여 설계한다.
일관성 유지하며 효율적으로 데이터를 관리하도록 설계한다.
불필요한 데이터와 데이터 중복이 발생하지 않도록 설계한다.
