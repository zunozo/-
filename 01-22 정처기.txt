서비스 지향 아키텍처 기반 어플리케이션 계층
표현층, 프로세스층, 비즈니스 논리 계층, 데이터 접근 계층

소프트웨어 상위 설계(시스템 아키텍처)
UI, interface, 전체적인 아키텍처
소프트웨어 아키텍처 설계 -> 모듈, 컴포넌트

모듈의 기본요소(함수max를 떠올리면됨)
입력요소 : (parameter 받는부분)
출력요소 : return
기능요소 : module의 정의
기관요소 : 알고리즘
내부자료요소 : 내부 변수

MVC 패턴
view : 뷰는 사용자가 사용할 결과물을 생성하기 위해 모델로부터 데이터를 가져온다.
controller : 데이터와 비즈니스 로직 사이의 상호 동작
model : 응용 프로그램의 데이터
컨트롤러가 직접 모델에서 정보를 가져올 수도 있음.

코드설계 단게 시 고려사항
이식성x 갱신이 용이, 추가시 확장이 용이

ISAM(색인 순차 편성)파일
직접 편성 파일(Random Accesss)보다 느리다.
인덱스를 추가한 인덱스 순차파일.
